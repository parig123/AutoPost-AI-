Frequently Asked Questions

Everything you need to know about PostGenius AI — from getting started to scheduling and LinkedIn publishing.

Getting Started
How do I get started with PostGenius AI?
Log in using your LinkedIn account via the Login page. Once authenticated, you'll be taken to the Dashboard where you can generate your first AI-powered post. Go to Create Post, enter a topic, pick a tone and length, and click Generate.
Do I need any API keys to use this?
Yes — you need a GROQ_API_KEY (free at console.groq.com) and LinkedIn OAuth credentials (from the LinkedIn Developer Portal). Both go in the .env file. The app will still start without them, but AI generation and LinkedIn posting won't work until they're added.
How do I run this on another computer?
  1. Install Python 3.10+ and pip
  2. Copy the project folder to the new machine
  3. Create a virtual environment: python -m venv venv
  4. Activate it and run: pip install -r requirements.txt
  5. Copy your .env file (never commit it to Git)
  6. Run: python app.py
AI Generation
What AI model powers the post generation?
PostGenius AI uses Llama 3.3 70B via the Groq API — one of the fastest and most capable open models available. Images are generated via Pollinations.AI (free, no key required).
Why is my post generation slow or failing?
This usually means your GROQ_API_KEY is missing, expired, or hitting the free rate limit (30 req/min). The app automatically retries up to 4 times on rate-limit errors. If it keeps failing, wait 60 seconds and try again, or upgrade your Groq plan.
Can I edit the AI-generated post before publishing?
Yes! On the Create Post page, after generation the text is fully editable. You can tweak wording, add personal stories, or change the tone before saving or publishing.
Scheduling & Publishing
Do I need to keep the browser open for scheduled posts?
No. The scheduler runs as a background thread inside the server process. As long as python app.py is running on your machine, posts will publish automatically at the scheduled time — even with the browser closed.
My post shows "Failed" — what do I do?
Common reasons:
1. Expired LinkedIn token — go to Settings and reconnect your account.
2. Empty post content — ensure generated_text is not blank.
3. LinkedIn API limit — personal accounts can post ~150 times/day.
Check linkedin_debug.log in the project folder for the exact error message.
Why does the site show "Connection Refused"?
This means the Flask server is not running. Open a terminal, navigate to the project folder, activate the virtual environment, and run python app.py. The site will be available at http://localhost:5000.
